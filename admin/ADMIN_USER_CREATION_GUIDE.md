# Admin User Creation Guide
## Uganda Martyrs University - Graduate Digital Clearance System

---

## 🎯 Overview
As a **System Administrator**, you have the ability to create **ALL types of user accounts** in the system, not just students. This includes:

---

## 👥 User Types You Can Create

### 1. 👨‍🎓 Students
- **Purpose:** Apply for graduation clearance
- **Email Requirement:** MUST contain `@stud` (e.g., `john@stud.umu.ac.ug`)
- **Additional Info Required:**
  - Registration Number (Required)
  - Course/Programme
  - Campus
- **Access:** Student dashboard, application status tracking

---

### 2. 💰 Finance Officer
- **Purpose:** Approve/reject financial clearances
- **Email Requirement:** Any valid email address
- **Additional Info Required:** None (optional phone number)
- **Access:** Finance dashboard, payment verification

---

### 3. 📚 Library Officer
- **Purpose:** Manage library book returns and clearances
- **Email Requirement:** Any valid email address
- **Additional Info Required:** None
- **Access:** Library dashboard, book return tracking

---

### 4. 💻 ICT Officer
- **Purpose:** Handle equipment returns and technical clearances
- **Email Requirement:** Any valid email address
- **Additional Info Required:** None
- **Access:** ICT dashboard, equipment tracking (laptops, damaged items)

---

### 5. 🎓 Faculty Dean
- **Purpose:** Academic approvals, results verification, dissertation approval
- **Email Requirement:** Any valid email address
- **Additional Info Required:** None
- **Access:** Dean dashboard, academic records

---

### 6. 📋 Academic Registrar
- **Purpose:** Final approval authority, graduation list management
- **Email Requirement:** Any valid email address
- **Additional Info Required:** None
- **Access:** Registrar dashboard, master approval table, PDF generation

---

### 7. 🔧 System Administrator
- **Purpose:** Full system access and user management
- **Email Requirement:** Any valid email address
- **Additional Info Required:** None
- **Access:** Admin dashboard, all system functions, user management

---

## 🚀 How to Create Users

### Step 1: Access User Creation
1. Login as Admin
2. Go to **Admin Dashboard**
3. Click **"User Management"** or **"Create New User"**

### Step 2: Fill User Information
1. **Full Name** - Required
2. **Email Address** - Required
   - Students: Must contain `@stud`
   - Others: Any valid email
3. **Phone Number** - Optional
4. **Password** - Required (user can change later)

### Step 3: Select User Role
1. Choose from dropdown:
   - Student
   - Finance Officer
   - Library Officer
   - ICT Officer
   - Faculty Dean
   - Academic Registrar
   - System Administrator

### Step 4: Additional Information
- **For Students Only:**
  - Registration Number (Required)
  - Course/Programme
  - Campus (Nkozi, Rubaga, Masaka, Ngetta, Fort Portal)

- **For Officers/Admins:**
  - No additional information required

### Step 5: Create Account
- Click **"Create User"** button
- User account is created and active immediately
- User can login with provided credentials

---

## ✅ Best Practices

### For Students:
✅ Always use `@stud` in email addresses
✅ Provide accurate registration numbers
✅ Select correct campus

### For Officers:
✅ Use official university email addresses when possible
✅ Use descriptive names (e.g., "Finance Officer - Main Campus")
✅ Set strong passwords initially

### For Administrators:
✅ Limit admin accounts (only create when necessary)
✅ Use personal university emails
✅ Document admin account creation in activity logs

---

## 🔐 Default Passwords

When creating accounts, you can set temporary passwords:
- **Students:** Recommended format: `Student@2026`
- **Officers:** Default: `officer123` (document for security team)
- **Admins:** Strong password required

**Important:** All users should change their passwords after first login!

---

## 📊 User Management After Creation

### View All Users
- Go to **User Management** page
- See all users with roles, statuses, and creation dates

### Edit Users
- Click **"Edit"** button next to any user
- Modify details, change roles, update information

### Deactivate Users
- Click **"Deactivate"** to temporarily disable accounts
- User cannot login but data is preserved

### Reset Passwords
- Use **"Reset Passwords"** function
- Generate new temporary passwords for users

## 🎓 Common Scenarios

### Scenario 1: New Student Registration
**Action:** Create Student account
**Requirements:** Email with @stud, Registration Number, Course, Campus
**Result:** Student can login and apply for clearance

### Scenario 2: New Finance Officer Hired
**Action:** Create Finance Officer account
**Requirements:** Name, Email, Password
**Result:** Officer can login and approve financial clearances

### Scenario 3: Dean for New Faculty
**Action:** Create Faculty Dean account
**Requirements:** Name, Email, Password
**Result:** Dean can login and approve academic records

### Scenario 4: Backup Registrar Needed
**Action:** Create Academic Registrar account
**Requirements:** Name, Email, Password
**Result:** Has access to final approvals and graduation list

### Scenario 5: New IT Admin Hired
**Action:** Create System Administrator account
**Requirements:** Name, Email, Strong Password
**Result:** Full system access for management

## ⚠️ Important Notes

1. **Email Validation:**
   - Students MUST have `@stud` in email
   - All other roles can use any email format

2. **Role Assignment:**
   - Roles are set at account creation
   - Can be changed later through "Edit User"

3. **Activity Logging:**
   - All user creation is logged
   - Includes who created the account and when

4. **Account Security:**
   - Set strong initial passwords
   - Advise users to change passwords on first login
   - Monitor admin account creation

5. **Data Integrity:**
   - Registration numbers must be unique for students
   - Email addresses must be unique across all users
   - Phone numbers are optional but recommended
